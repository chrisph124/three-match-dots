Game Architecture (layered)

Game corepure TS · grid, chain validation, loop detection, gravity/refill, scoringVitest-tested
RenderSkia — draws dots, active link path, particleson-device QA
InputGesture Handler worklet — touch → cell → chainon-device QA
EffectsReanimated + Skia — pop, fall-bounce, burst, shake, comboon-device QA
Meta UIexpo-router screens — title, HUD, game-over, settingson-device QA

Performance principle: keep gesture + hit-test + animation on the UI thread (worklets) to dodge the JS↔native bridge — the classic cause of RN game jank.

Watch Out

Honest caveats

  • Vitest tests RN-free code only. Core = unit-tested; Skia/Reanimated/gesture layers = on-device QA. "We have tests" ≠ "UI tested".
  • Analytics = compliance overhead. App Store Privacy Labels + Google Play Data Safety disclosures (PostHog keeps it minimal).
  • Android low-end perf. Test on a cheap real Android device before the Play release.
  • OTA is JS-only. Native changes (new native module) still need a full store build.
  • Keys out of git. Sentry/PostHog keys via EAS secrets / env vars.
  • Strict no-any escape hatch. Untyped 3rd-party libs → allowed only via inline eslint-disable + justification comment.
